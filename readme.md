Spatio-Directional Tree Viewer is a program which allows the navigation, visualization, and analysis of spatio-directional tree dumps generated by Rath's extension of Müller's path guiding algorithm, or by the BTC structure I implemented.

The visualizer has only been tested on modern versions of Linux Mint. Therefore, the following setup commands are for Linux-based operating systems. The application should run fine on other operating systems, assuming the correct installs.  

There is a submodule which contains some example `.sdt` dumps to experiment with. Run 
`git submodule update --init --recursive`
to clone these alongside the repository. 

# Setup

To set up the visualizer, ensure you have the following installed on your system:

- C++20 or later, and an associated compiler
- Cmake 3.16+

Alongside the following packages:

- OpenGL 3.3+ (core profile)
- GLFW 3.3+
- GLEW 2.0+
- Eigen3 3.3+

The following command should install the latest versions of the above packages:
`sudo apt install cmake build-essential libglfw3-dev libglew-dev libeigen3-dev libgl-dev`

Dear ImGui is fetched automatically by CMake during configuration, so it does not need to be installed manually.

To run the program, use the format

`./visualizer INPUTS`

where INPUTS is a comma-separated list of inputs that is either: 

A direct path to single dump
`<path-to-dump>`

or a list of arbitrary size with a sequence of dumps of the format

`[<path-to-dump-file><sequence-prefix>00, 
<path-to-dump-file><sequence-prefix>01, 
<path-to-dump-file><sequence-prefix>02]
`
or its compacted version 

`[<path-to-dump-file><sequence-prefix>[00, 01, 02]]`

For the visualizer to properly parse the files, they must contain a shared prefix. 
The dumps do not have to start with the first iteration, nor include each iteration, but must be provided in order from lowest resolution to highest resolution. 
The end of subsequent files must be increasing natural numbers.
The spaces are optional but the commas and encasing [] are mandatory.

If a dump is from the BTC structure, it must have "btc" somewhere in the filename (case-insensitive).


For example, assuming all the files exist and are valid dumps in sequence, the following is valid:

`./visualizer ../r0cketship-dump-btc.sdt, [../kitchen-vanilla-2047-dump-01.sdt, ../kitchen-vanilla-2047-dump-04.sdt, ../kitchen-vanilla-2047-dump-05.sdt, ../kitchen-vanilla-2047-dump-06.sdt, ../kitchen-vanilla-2047-dump-08.sdt], ../dump-outside-scene.sdt, [../kitchen-btc-511-dump-02.sdt, ../kitchen-btc-511-dump-05.sdt, ../kitchen-btc-511-dump-06.sdt]`

# Features

![The visualizer interface](vis-interface.png)

The left panel contains settings; 
the centre panel visualizes the spatial deposits as a point cloud; 
the right panel visualizes the directional components;
the top bar lists all loaded SDT trees and their iterations.

## Basic Controls

- Scroll to zoom
- Click to rotate
- Right click to pan

The loaded sequences are at the top and in the right panel. 
Click on a sequence in the top bar to switch the loaded structure.

Click in the spatial domain to select a cell. 
The directional heatmap of the cell is rendered on the right panel. 

## Basic Options

The left panel allows for filtering options. These include: 

- spatial filtering
- selecting cells based on their coordinate position
- changing the rendering space of the directional structures
- generation of the `.xml` file to create a spherical camera from the perspective of the selected spatial cell

It also shows statistics on the number of spatial cells in the scene, in the selected region, and the coordinates of any selected cell.
Select the show bounding boxes option to see the bounding boxes for each spatial cell in the selected region. 

The right panel allows for the exporting of any heatmap as a `.pfm` file, and for the creation of a KL map between any two heatmaps currently rendered.
For directional structures, it shows the radiance stored in any specific cell, and the integral of the PDF it defines.
For BTC structures it permits the navigation of the different tilings


## Modes

There are three selectable modes, plus one hidden mode, each with different features and settings. Some features are only accessible in certain modes. 

### Normal Mode

This is the default mode. You can right-click to select a second cell.

### Total Diff

This mode allows the comparison between structures within the same sequence. First, select the relevant sequence. Second, select dumps from that sequence. 
Now, any selections made will show all the children from subdivisions in the later sequence.
Select show heatmaps of all children to render all the children in the visualizer, with a maximum of `8` heatmaps in the visualizer. 

![Heatmaps of the child cells of a selection](vis-child-cells.png)

### Partial Diff

This mode allows the comparison between structures from different sequences. 
Select up to 7 structures from the left panel.
Then, any selection in the spatial domain will highlight the cell containing the same location. 


### Iteration Diff (Hidden)

This mode is not listed in the modes. To enter it, select one spatial cell from the top list and right-click-select a second cell. This will show, in the top bar, how many cells were created or pruned between the iterations.
